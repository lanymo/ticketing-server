# 부하테스트 시나리오

## 1. 테스트 환경

### 도구
- **k6** - 주요 부하 테스트 (JavaScript 기반, 시나리오 작성 용이)
- **JMeter** - 보조 테스트 (GUI 기반 복잡한 흐름 검증)
- **Gatling** - 리포트 생성 (Scala DSL, 상세 리포트)

### 테스트 환경 구성
```
[k6 실행 머신] → [Load Balancer] → [API Gateway] → [각 서비스]
                                                   → [Redis]
                                                   → [Kafka]
                                                   → [RabbitMQ]
                                                   → [MySQL]
```

### 성능 목표 (Pass/Fail 기준)

| 지표 | 목표 | 실패 기준 |
|------|------|-----------|
| TPS | 1,000 req/s | < 800 req/s |
| 응답 시간 p95 | < 1.5초 | > 3초 |
| 응답 시간 p99 | < 2초 | > 5초 |
| 에러율 | < 1% | > 5% |
| 티켓 중복 발급 | 0건 | 1건 이상 |
| 재고 초과 발급 | 0건 | 1건 이상 |

### S1. 정상 부하 테스트 (Smoke → Load)

**목적**: 평상시 트래픽에서 기본 기능이 안정적으로 동작하는지 확인

**부하 프로파일**
```
VU
1000 |          ┌─────────────────────┐
 500 |      ┌───┘                     └───┐
   0 └──────┴─────────────────────────────┴──→ time
       0    2m        12m             22m  24m
```

**k6 스크립트**
```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';
import { Trend, Counter } from 'k6/metrics';

const reservationTime = new Trend('reservation_duration');
const reservationErrors = new Counter('reservation_errors');

export const options = {
  stages: [
    { duration: '2m',  target: 500  },  // Ramp-up
    { duration: '10m', target: 1000 },  // Steady state
    { duration: '2m',  target: 0    },  // Ramp-down
  ],
  thresholds: {
    http_req_duration: ['p(95)<1500', 'p(99)<2000'],
    http_req_failed:   ['rate<0.01'],
    reservation_duration: ['p(95)<1500'],
  },
};

const BASE_URL = 'http://api-gateway:8080';

export function setup() {
  // 테스트용 공연 생성 (사전 데이터 세팅)
  const res = http.post(`${BASE_URL}/api/events`, JSON.stringify({
    name: '부하테스트 공연',
    totalSeats: 100000,
    openAt: new Date().toISOString(),
  }), { headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer ADMIN_TOKEN' } });
  return { eventId: res.json('id') };
}

export default function (data) {
  const userId = `user_${__VU}_${__ITER}`;

  // Step 1: 로그인
  const loginRes = http.post(`${BASE_URL}/api/users/login`, JSON.stringify({
    username: userId, password: 'test1234'
  }), { headers: { 'Content-Type': 'application/json' } });

  check(loginRes, { 'login success': (r) => r.status === 200 });
  const token = loginRes.json('accessToken');
  const headers = { Authorization: `Bearer ${token}`, 'Content-Type': 'application/json' };

  // Step 2: 공연 정보 조회
  const eventRes = http.get(`${BASE_URL}/api/events/${data.eventId}`, { headers });
  check(eventRes, { 'event fetched': (r) => r.status === 200 });
  sleep(1);

  // Step 3: 좌석 목록 조회
  const seatsRes = http.get(`${BASE_URL}/api/events/${data.eventId}/seats?status=AVAILABLE`, { headers });
  check(seatsRes, { 'seats fetched': (r) => r.status === 200 });
  const seats = seatsRes.json('seats');
  if (!seats || seats.length === 0) return;

  // Step 4: 예매 요청
  const seatId = seats[Math.floor(Math.random() * seats.length)].id;
  const startTime = Date.now();
  const reserveRes = http.post(`${BASE_URL}/api/reservations`, JSON.stringify({
    eventId: data.eventId, seatId: seatId,
  }), { headers });

  reservationTime.add(Date.now() - startTime);
  const success = check(reserveRes, {
    'reservation success or conflict': (r) => r.status === 201 || r.status === 409,
  });
  if (!success) reservationErrors.add(1);

  sleep(2);
}
```

**검증 항목**
- [ ] p95 응답 시간 < 1.5초
- [ ] 에러율 < 1%
- [ ] MySQL 커넥션 풀 고갈 없음
- [ ] Redis 메모리 누수 없음

### S2. 티켓 오픈 스파이크 테스트

**목적**: 티켓 오픈 순간 수만 명이 동시 접속할 때 시스템이 버티는지 검증

**부하 프로파일**
```
VU
50000|              ██
     |             ████
10000|        ████████████
 1000|    ████████████████████
    0└────┴────┴────┴────┴────┴──→ time
      0   1m   2m   3m   5m  10m
```

**k6 스크립트**
```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';
import { SharedArray } from 'k6/data';

// 사전 생성된 유저 토큰 목록 로드
const users = new SharedArray('users', function () {
  return JSON.parse(open('./users_50000.json')); // [{ userId, token }, ...]
});

export const options = {
  scenarios: {
    spike: {
      executor: 'ramping-arrival-rate',
      startRate: 100,
      timeUnit: '1s',
      preAllocatedVUs: 50000,
      maxVUs: 60000,
      stages: [
        { duration: '30s', target: 100   },  // 대기
        { duration: '10s', target: 50000 },  // 오픈 순간 스파이크
        { duration: '2m',  target: 10000 },  // 안정화
        { duration: '3m',  target: 1000  },  // 감소
        { duration: '2m',  target: 0     },
      ],
    },
  },
  thresholds: {
    http_req_duration: ['p(99)<5000'],    // 스파이크 시 p99 5초 허용
    http_req_failed:   ['rate<0.05'],     // 에러율 5% 이하
  },
};

const BASE_URL = 'http://api-gateway:8080';
const EVENT_ID  = __ENV.EVENT_ID;

export default function () {
  const user = users[__VU % users.length];
  const headers = { Authorization: `Bearer ${user.token}`, 'Content-Type': 'application/json' };

  // 대기열 진입
  const queueRes = http.post(`${BASE_URL}/api/queue/enter`, JSON.stringify({
    eventId: EVENT_ID,
  }), { headers });

  check(queueRes, { 'queue entered': (r) => r.status === 200 || r.status === 202 });

  if (queueRes.status !== 200) return;

  // 대기 순번 폴링 (최대 10회)
  let admitted = false;
  for (let i = 0; i < 10; i++) {
    sleep(1);
    const statusRes = http.get(`${BASE_URL}/api/queue/status?eventId=${EVENT_ID}`, { headers });
    if (statusRes.json('status') === 'ADMITTED') {
      admitted = true;
      break;
    }
  }

  if (!admitted) return;

  // 입장 후 예매
  const reserveRes = http.post(`${BASE_URL}/api/reservations`, JSON.stringify({
    eventId: EVENT_ID,
    seatId: Math.floor(Math.random() * 10000) + 1,
  }), { headers });

  check(reserveRes, {
    'reserved or conflict': (r) => r.status === 201 || r.status === 409,
    'not 500': (r) => r.status !== 500,
  });
}
```

**검증 항목**
- [ ] 스파이크 구간 에러율 < 5% (429 Too Many Requests 포함 허용)
- [ ] 스파이크 해소 후 정상 응답 복귀 < 30초
- [ ] Redis 대기열 데이터 유실 없음
- [ ] OOM, GC 과부하 없음 (JVM 힙 모니터링)
- [ ] Kafka Consumer Lag 5분 이내 해소

### S3. 대기열 처리 테스트

**목적**: 대기열 순서 보장, 입장 토큰 발급 정확성, 처리 속도 검증

**시나리오 흐름**
```
30,000명 동시 대기열 진입
    │
    ▼
Scheduler: 1초마다 100명씩 입장 허용
(예상 전체 처리 시간: 300초)
    │
    ▼
입장 순서가 진입 순서와 일치하는지 검증
    │
    ▼
입장 토큰 중복 발급 여부 확인
```

**k6 스크립트**
```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';
import { Counter, Gauge } from 'k6/metrics';

const outOfOrderAdmissions = new Counter('out_of_order_admissions');
const currentQueueSize = new Gauge('queue_size');

export const options = {
  scenarios: {
    queue_test: {
      executor: 'shared-iterations',
      vus: 30000,
      iterations: 30000,
      maxDuration: '10m',
    },
  },
};

const BASE_URL = 'http://api-gateway:8080';
const EVENT_ID  = __ENV.EVENT_ID;

export default function () {
  const userId = `user_${__VU}`;
  const token  = getToken(userId);
  const headers = { Authorization: `Bearer ${token}`, 'Content-Type': 'application/json' };

  // 대기열 진입 (진입 시각 기록)
  const enterTime = Date.now();
  const queueRes  = http.post(`${BASE_URL}/api/queue/enter`,
    JSON.stringify({ eventId: EVENT_ID }), { headers });

  check(queueRes, { 'entered queue': (r) => r.status === 200 });
  const myRank = queueRes.json('rank');

  // 대기열 크기 모니터링
  const queueStatus = http.get(`${BASE_URL}/api/queue/info?eventId=${EVENT_ID}`, { headers });
  currentQueueSize.add(queueStatus.json('totalWaiting'));

  // 입장 대기 폴링
  let admitTime = null;
  let myAdmitOrder = null;

  for (let i = 0; i < 600; i++) {  // 최대 10분 대기
    sleep(1);
    const statusRes = http.get(`${BASE_URL}/api/queue/status?eventId=${EVENT_ID}`, { headers });
    const body = statusRes.json();

    if (body.status === 'ADMITTED') {
      admitTime    = Date.now();
      myAdmitOrder = body.admitOrder;
      break;
    }
  }

  if (!admitTime) {
    console.error(`VU ${__VU}: 입장 타임아웃`);
    return;
  }

  // 순서 검증: 내 rank와 실제 입장 순서 비교 (허용 오차 ±10)
  if (Math.abs(myAdmitOrder - myRank) > 10) {
    outOfOrderAdmissions.add(1);
    console.warn(`순서 이탈 - rank: ${myRank}, admitOrder: ${myAdmitOrder}`);
  }
}

function getToken(userId) {
  const res = http.post(`${BASE_URL}/api/users/login`,
    JSON.stringify({ username: userId, password: 'test1234' }),
    { headers: { 'Content-Type': 'application/json' } });
  return res.json('accessToken');
}
```

**검증 항목**
- [ ] 대기열 순서 이탈 건수 = 0 (허용 오차 ±10 이내)
- [ ] 입장 토큰 중복 발급 = 0건
- [ ] TTL 만료 후 재입장 불가 처리 정상 동작
- [ ] 30,000명 전원 입장 완료 (유실 없음)

### S4. 동시성 제어 테스트 (핵심)

**목적**: 동일 좌석에 동시 요청이 몰릴 때 1명만 예매 성공하는지 검증

**시나리오**
```
좌석 1개 (seatId=1, 재고=1)
    │
    ▼
5,000명이 동시에 동일 좌석 예매 요청
    │
    ▼
기대 결과: 성공 1건, 실패(409) 4,999건
         재고 초과 발급 = 0건
```

**k6 스크립트**
```javascript
import http from 'k6/http';
import { check } from 'k6';
import { Counter } from 'k6/metrics';

const successCount   = new Counter('reservation_success');
const conflictCount  = new Counter('reservation_conflict');
const errorCount     = new Counter('reservation_error');

export const options = {
  scenarios: {
    concurrency_test: {
      executor: 'shared-iterations',
      vus: 5000,
      iterations: 5000,
      maxDuration: '30s',
    },
  },
  thresholds: {
    reservation_success: ['count<=1'],   // 성공은 반드시 1건 이하
  },
};

const BASE_URL = 'http://api-gateway:8080';
const EVENT_ID  = __ENV.EVENT_ID;
const SEAT_ID   = 1; // 동일 좌석

export default function () {
  const token   = getPreloadedToken(__VU);
  const headers = { Authorization: `Bearer ${token}`, 'Content-Type': 'application/json' };

  const res = http.post(`${BASE_URL}/api/reservations`, JSON.stringify({
    eventId: EVENT_ID,
    seatId:  SEAT_ID,
  }), { headers, timeout: '10s' });

  if (res.status === 201)      successCount.add(1);
  else if (res.status === 409) conflictCount.add(1);
  else                         errorCount.add(1);

  check(res, {
    'valid response': (r) => r.status === 201 || r.status === 409,
    'no server error': (r) => r.status !== 500,
  });
}
```

```sql
HAVING cnt > 1;
-- 결과: 0 rows (초과 발급 없어야 함)

-- Redis 재고 확인
-- REDIS: GET stock:{eventId}:1

```

**검증 항목**
- [ ] 예매 성공 건수 = 정확히 1건
- [ ] DB 중복 예약 레코드 = 0건
- [ ] Redis 재고 음수 없음 (0 이상 유지)

### S7. 결제 실패 복구 테스트

**목적**: 결제 실패 시 재고가 정확히 롤백되고 해당 좌석이 재예약 가능한지 검증

**시나리오 흐름**
```
1,000명 예매 성공 (임시 예약 상태)
    │
    ▼
PG사 Mock 서버를 강제 실패 모드로 전환
    │
    ▼
500명 결제 실패 처리
    │
    ▼
검증: 실패한 500석의 재고가 Redis + DB 모두 복구되었는지
    │
    ▼
다른 1,000명이 복구된 좌석 재예매 시도
    │
    ▼
검증: 재예매 성공 = 500건 (복구된 좌석 수만큼)
```

**k6 스크립트**
```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  scenarios: {
    // Phase 1: 예매
    reserve: {
      executor: 'shared-iterations',
      vus: 1000, iterations: 1000,
      maxDuration: '2m',
      tags: { phase: 'reserve' },
    },
    // Phase 2: 복구 후 재예매 (Phase 1 완료 후 실행)
    re_reserve: {
      executor: 'shared-iterations',
      vus: 1000, iterations: 1000,
```
